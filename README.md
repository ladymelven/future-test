Тестовое задание для Future Group (ТЗ находится здесь https://github.com/fugr-ru/frontend-javascript-test).

Компонент выгружает с сервера массив данных (имена и контактные данные) - по выбору
пользователя маленький (32 элемента) или большой (1000 элементов) - и строит на
его основе таблицу. Строки таблицы можно сортировать по любому полю (по возрастанию
и убыванию значений) по клику на шапку, фильтровать по ключевому слову, добавлять
новые. По клику на строку под таблицей выводится информация об элементе (адрес и описание).

При длине массива больше 50 таблица бьется на страницы. Модуль пагинации находится
под таблицей и выводится только для многостраничных таблиц. Показывает текущую страницу +
две по бокам; если расстояние от первой и/или последней страницы больше 2, то вставляются
заглушки вида "..." (общий вид 1 ... 4 5 6 ... 20). Текущая страница выделена жирным.

Компонент включает в себя следующие модули: модуль выбора размера массива, индикатор загрузки
(выводится, пока грузятся данные), собственно таблицу, строку таблицы, пагинацию,
формы фильтрации и добавления новой строки, модуль с расширенной информацией о строке.
Все сделано вручную, кроме CSS индикатора (взят с https://projects.lukehaas.me/css-loaders/).
Индикатор загрузки и пагинацию можно заново использовать без дополнительной настройки.

Компонент создавался с помощью Create React App. Никаких других зависимостей нет,
верстка ручная. В качестве devDependencies - jest и enzyme с соответствующими плагинами
для Реакта. Для запроса на сервер можно было бы использовать axios (в проекте
сделано через fetch API), но запрос делается только в одном месте, поэтому смысла в
дополнительной зависимости я не вижу.

В форме добавления строки проверяется только заполнение всех полей. Для поля
email работает нативная валидация браузера по типу ввода. Id в полученном с сервера
массиве уже не являются уникальными, поэтому смысла проверять уникальность id нет.
При добавлении новой строки не предусмотрено заполнение адреса и описания (см. тз по ссылке),
при отправке формы добавляются заглушки. В качестве улучшения имеет смысл добавить валидацию
номера телефона и улучшить валидацию email. Можно использовать yup или сделать руками.

Компонент протестирован вручную и с помощью юнит-тестов на Jest и Enzyme (рендеринг
компонентов, кроме самых простых, и eventHandlers в корневом компоненте). Измерено
с помощью performance.now() время выполнения всех потенциально проблемных операций
(в целом не превышает 10мс, время рендеринга при загрузке большого объема 35мс).
